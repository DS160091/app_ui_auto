# appcard_gen — App 操作手册生成器（automobile 子 skill）

从一组页面截图，自动识别每页的可交互/可断言元素，套 `TEMPLATE.md` 骨架生成一份 `app_card.md`（页面/控件操作手册），写到配置指定的位置。

它造的是**手册**，本身不是回归流程：不进 `businessLine/` 业务树、不登记 `routes.yaml`。手册供后续跑流程时当「这个 App 长啥样、控件怎么定位」的参考。

## 何时用

要给一条回归流程接入一个**之前没出现过的 App**（UI 完全不同，无法复用已有手册）时，用本 skill 从截图自动生成该 App 的控件手册，免去逐页人肉标注坐标/文案/定位方式。

## 输入：配置文件驱动 + 执行时强校验

调用方式：本 skill 是 automobile 的**子 skill**，不再是独立 `/appcard_gen` 命令。在 `/automobile` 会话中明确要求「生成 appcard」并给出配置路径即触发，由路由层读取并遵循本文件执行。**用户提前写好配置文件**，执行时把配置路径给我。我先强校验，校验不过立即中断、报错，**绝不生成半成品**。

1. 用户复制 `appcard_gen.config.yaml` 为自己的配置，填好字段（见样例注释）。配置字段：

   | 字段 | 必填 | 说明 |
   |---|:--:|---|
   | `screenshots_dir` | 是 | 截图目录，每个关键页面/弹窗各一张，建议序号命名 |
   | `output_path` | 是 | 生成的 `app_card.md` 写到哪，父目录自动创建 |

   相对路径相对配置文件所在目录解析。只有两项必填——其余能从输出文件名/首图尺寸/adb 推出，填了更准。

2. **强校验（硬约束）**：执行 `python3 <skill_dir>/validate.py <config.yaml>`。
   - 退出码 **0** 且 `ok: true` → 拿 JSON 里的 `config` / `screenshots` / `abs_output_path` / `app_name` 继续生成。
   - 退出码 **1** 或 `ok: false` → **立即停止**，把 `errors` 原样告诉用户，要求改好配置再来。**不得猜测、不得跳过、不得用默认值硬跑。**
   - 有 `warnings` → 复述给用户但可继续。

## 生成流程

校验通过后逐步执行：

1. **逐图识别**：对截图目录里每张图（用 `pages` 的语义说明辅助），用视觉能力识别本页所有可交互/可断言元素，逐个提取：
   - 元素名称、类型（按钮/Tab/输入框/弹窗/图标/文本…）
   - 文本或图标特征（按钮文案、placeholder、标题）
   - 估算坐标 `x, y, w, h`（基于 `reference_resolution`，未配置则按首图实际尺寸）
   - 定位建议（见下方「定位优先级」）
2. **套模板拼装**：以 `TEMPLATE.md` 为骨架，把每页填成「页面结构 + 关键元素表 + 自动化备注」一节；补齐开头的基本信息/操作节奏/全局通用结构，和结尾的定位策略/易误点/建议断言点。
3. **写出**：写到 `abs_output_path`。父目录不存在先创建。
4. **回读校对**：对照截图复核一遍重复文案、同名按钮、悬浮控件等易误点是否标注到位（见下）。
5. 向用户报告：生成位置、覆盖了几页、标了哪些易误点。

## 质量要求（决定后续跑得稳不稳）

识别与成文必须满足以下几点，否则生成的手册会误导 UI 模型——这是本 skill 最值钱的部分：

- **定位优先级**：① 文本定位 > ② 文本+区域限定（重复文案）> ③ 唯一值（如单号/编号）精确定位 > ④ 相对坐标（无文本图标）> ⑤ 图像特征（悬浮按钮）。每个元素给出最稳的那种。
- **易误点必标**：
  - 多个弹窗都有「确认/取消」→ 必须结合弹窗标题/文案限定范围，杜绝误点背景同名主按钮。
  - 列表里高度相似的卡片 → 用唯一值（如单号/编号）定位，不能只靠标题名。
  - 同一文案在页面多处出现 → 标明用父区域区分。
  - 录屏/答疑等悬浮控件 → 标明忽略，别当业务控件。
- **坐标只作辅助**：坐标随分辨率/版本漂移，正文里坐标永远是「估算」，定位建议优先给文本/特征。
- **覆盖弹窗**：弹窗是漏标重灾区，每个会弹的窗都要单独成节。

## 与 automobile 的关系

本 skill 位于 `automobile/sub_skills/appcard_gen/`，是 automobile 的子 skill（**非顶层 `/` 命令**），仅在用户于 `/automobile` 会话中明确要求生成手册时触发，不参与常规回归诉求的路由匹配、不登记 `routes.yaml`。生成的手册通常写到某条回归流程叶子的 `app_cards/<app>.md`（由 `output_path` 决定），供 automobile 跑流程时使用——见 automobile 新增流程指南第 2 步。
